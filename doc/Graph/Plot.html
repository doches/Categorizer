Graph::Plot

Abstract class from which data-specific plots inherit. Sets some default options for GNUPlot (such as terminal type and output) so as to unify resulting graphs.

Could do with a bit more customization, so plots are even more homogenous.

Public Class Methods

new(output) click to toggle source

Create a new Plot. ‘output’ is used to name the resulting file, regardless of term type (e.g. resulting graph will be in “output.”). GNUPlot variables can be accessed via has syntax, as in the example.

Example usage (won’t work, because by default Plot doesn’t know about any data). 

  plot = Plot.new("foo")
  plot.xrange = "[0:19]"
  plot.render

# File lib/Graph.rb, line 23
    def initialize(output)
      @tmpdir = "reports"
      @output = output
      @attribs = {
        "terminal" => "postscript",
        "size" => "1,1",
        "output" => "\"#{output}.eps\"",
        "xrange" => "[0:9]",
        "yrange" => "[0:1]"
      }
    end

method_missing(sym,*args,&blk) click to toggle source

Beautiful hack that lets you access GNUPlot options (terminal, size, output, xrange, etc.) via hash syntax. 

# File lib/Graph.rb, line 37
    def method_missing(sym,*args,&blk)
      if sym.to_s =~ /^(.+)=$/
        @attribs[$1] = args[0]
      elsif @attribs.include?(sym.to_s)
        return @attribs[sym.to_s]
      end
      return nil
    end

to_plt(custom_text = "") click to toggle source

Write current attributes/data into a .plt file. Called by ‘render’. ‘custom_text’ is added to the end of the file, after all attributes have been set. Add references to data files here. 

# File lib/Graph.rb, line 57
    def to_plt(custom_text = "")
      file = "#{@tmpdir}/#{@output}.plt"
      fname = file
      file = File.open(file,"w")
      @attribs.each_pair do |key,value|
        file.puts "set #{key} #{value}"
      end
      file.puts custom_text
      file.close
      return fname
    end
